# Product Management — Class Notes
## March 23, 2026 — Week 9

**Topic:** Social app / dating app concept — gamification + common interests to reduce repetitive conversations; user journey; personas; interaction model; onboarding stages

---

I. Problem framing (from interviews + team discussion)
   A. Core friction
      1. Repetitive conversation openers (same questions)
      2. Low effort / low energy interactions lead to drop-off
      3. Hard to reach deeper conversation after first contact
   B. Opportunity
      1. Use structured activities to break ice
      2. Reduce “small talk only” and increase meaningful connection

II. App concept — matching + interaction design
   A. Gamification as ice-breaker
      1. Group activities + games (shared action)
      2. Designed to start connection with strangers
   B. Shared interest hooks
      1. COMMON INTEREST feature
      2. BUCKET LIST feature (shared goals / activities to do together)
   C. Quality over quantity interaction model
      1. Limit interactions / encourage deeper engagement per day
      2. Fewer interactions -> more intentional conversation

III. Conversation mechanics (reduce repetition + go deeper)
   A. “UNFILTERED” Q&A model (shared answering)
      1. Both people answer the same question
      2. Answer first to make the other person comfortable
   B. Deep question prompts
      1. Use conversation starters beyond shallow greetings
      2. Example style: “WOULD YOU RATHER” type prompts
   C. Compatibility through signals
      1. Personality test / analysis as matchmaking input
      2. Value alignment + “deal breakers” surfaced early

IV. Profiles + updates (avoid static profiles)
   A. Profile content
      1. Interests / what you value
      2. What you cannot stand / deal breakers
      3. Personality test results used for conversation start
   B. Profile refresh cycle
      1. Update weekly / every 2 weeks / monthly (as discussed)
      2. Keep match inputs aligned with current interests

V. Customer journey (stages + touchpoints)
   A. Stages (high-level)
      1. Awareness (how users realize they need to meet/connect)
      2. Discovery (how users find people/events)
      3. First contact (initial interaction via activity / messages)
      4. Evaluation (shared values + “vibe”)
      5. Follow-up (move from first contact to repeated connection)
      6. Relationship building (repeat interactions OR transition out)
   B. Example touchpoints (as described)
      1. Digital: profiles, matching, message prompts, conversation starters
      2. Physical: group sessions, classes, restaurants / meetups

VI. Personas + segmentation logic (who this fits best)
   A. Persona basis (from team summary)
      1. Prefer organizers / structured group contexts
      2. Prefer group projects / classroom / scheduled settings
   B. Persona selection into features
      1. Use values + activity preference to decide the interaction mode
      2. Match where the user feels comfortable (reduce anxiety)

VII. Recommendations prioritization (feature voting + next steps)
   A. Feature shortlist
      1. Common interest / shared interest recommendation
      2. Quality conversation starters (deep prompts + Q&A)
   B. Personalization strategy (as discussed)
      1. Make “what to do next” and “what to talk about next” more visible
      2. Guide interaction using prompts + activity suggestions
   C. Measurement idea (team discussion)
      1. Increase meaningful conversation depth
      2. Reduce repetitive interaction loops over time
